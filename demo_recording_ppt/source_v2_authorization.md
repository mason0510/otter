## 第三部分：创建授权（核心亮点！）(1分30秒)

### 步骤 1: 进入授权管理
1. 点击页面顶部 "**授权管理**" 按钮
2. 看到授权状态：**未授权**

### 步骤 2: 创建授权
1. 点击 "**创建授权**" 按钮
2. 进入授权配置页面

### 步骤 3: 配置授权参数
**展示配置界面**（默认值）：
```
Token 类型: SUI
每日限额: 0.1 SUI
单笔限额: 0.1 SUI
有效期: 30 天
```

**解说词**：
> "授权模式的核心是安全可控：设置每日限额和单笔限额，防止误操作或恶意调用。有效期 30 天，可随时撤销。"

### 步骤 4: 签名创建授权
1. 点击 "**创建授权**" 按钮
2. Sui Wallet 弹出签名请求
3. **重点展示**：签名内容（create_authorization 函数）
4. 点击 "Approve"
5. ✅ 授权创建成功！

**展示结果**：
- 🛡️ 授权状态：**已启用**
- 📋 授权对象 ID: `0xd078...aeca`
- 📊 已用额度: 0 / 0.1 SUI

**解说词**：
> "签名一次，创建了一个链上授权对象。这个对象包含我的限额规则，后续转账和 Swap 都会使用这个授权，无需重复创建。"

## 第四部分：授权模式转账 (1分钟)

### 场景 1: 第一次转账（使用授权）

**输入指令**：
```
转入0.001到0x6c90dc192d728b32c20e34336137775bf632e51035d784dfea0df73c3aaba225
```

**操作流程**：
1. 在输入框输入上述指令
2. 点击 "**开始执行**"
3. **重点展示**：交易摘要区域

**关键画面**：
```
🛡️ 授权模式
此交易将使用授权对象执行，无需重复签名
```

4. 点击 "**确认并签名**"
5. Sui Wallet 弹出签名
6. ✅ 交易成功！显示 toast 提示

**解说词**：
> "注意看，页面显示'授权模式'提示。这意味着交易会使用我刚才创建的授权对象。虽然仍需签名（区块链安全要求），但不需要再创建新的授权。"

### 场景 2: 第二次转账（验证授权复用）

**输入指令**：
```
转入0.002到0x6c90dc192d728b32c20e34336137775bf632e51035d784dfea0df73c3aaba225
```

**操作流程**：
1. 再次输入转账指令（金额改为 0.002）
2. 点击 "开始执行"
3. **强调**：仍然显示 "🛡️ 授权模式"
4. 签名执行
5. ✅ 交易成功

**查看授权状态**（切换到授权管理页面）：
- 📊 已用额度: 0.003 / 0.1 SUI
- ✅ 授权对象仍然有效

**解说词**：
> "第二次转账，授权对象自动复用，额度更新为 0.003 SUI。这就是授权模式的价值：一次创建，持续使用。"

## 第六部分：授权管理功能 (1分钟)

### 功能 1: 查看授权详情
1. 打开授权管理页面
2. **展示**：
   - 授权状态：✅ 已启用

3. 授权状态恢复为 ✅ 已启用

1. 点击 "**断开授权**"
2. 清除本地授权对象 ID
3. 页面显示：**未授权**

**解说词**：
> "授权完全在您的控制之下：可以随时禁用、启用或完全断开。安全性和便捷性兼得。"

## 常见问题

### Q: 授权模式是否真的免签？
A: **不是**。每笔交易仍需签名（区块链安全要求）。授权模式的价值是：
- 不需要重复创建授权
- 限额保护机制
- 可随时撤销

### Q: 如果授权对象丢失怎么办？
A: 点击 "断开授权"，重新创建即可。旧授权对象会自然过期。

### Q: 授权对象可以共享吗？
A: 当前版本仅支持创建者使用。未来可扩展为团队共享授权。

### Q: 录制时交易失败？
A: 检查钱包余额、网络连接、Gas 费是否充足。
